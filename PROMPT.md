# 작업 지시서: Sangil's Times Table Quest (구구단 암기 테스트 웹앱)

아래 사양대로 **단일 파일 `index.html`** 하나로 완성하라. 완성 후 브라우저에서 열기만 하면 작동해야 한다 (서버·빌드·외부 라이브러리·인터넷 연결 불필요).

## 0. 사용자
- 상일(Sangil), 2017년생 9세, **영어 사용자** → 화면의 모든 텍스트와 음성은 영어.
- 목표: **구구단(2~9단) 완전 암기**. 하루 4~5회 시험 볼 예정.
- 스마트폰/태블릿에서도 쓸 수 있게 모바일 우선 레이아웃 (큰 버튼, 터치 친화).

## 1. 기술 스택 (변경 금지)
- 단일 `index.html`: 순수 HTML + CSS + Vanilla JS. 프레임워크·CDN·외부 폰트·외부 이미지·외부 오디오 파일 금지.
- 소리:
  - 효과음: **Web Audio API**로 직접 합성 (정답 = 밝은 상승 아르페지오 "딩동!", 오답 = 부드러운 낮은 톤. 오답음은 무섭거나 불쾌하지 않게).
  - 음성: **SpeechSynthesis API** (`lang: 'en-US'`)로 칭찬/격려 문구를 실제로 말해준다.
  - 화면에 🔊 음소거 토글 버튼 제공 (상태는 localStorage에 저장).
- 데이터 저장: **localStorage** (문제별 통계, 시험 기록, 설정).

## 2. 시험 규칙 (핵심)
- **1회 시험 = 20문제**, 범위는 2×1 ~ 9×9 (총 72개 사실).
- **출제 우선순위**: localStorage의 문제별 기록을 보고
  1. 최근에 틀린 적 있는 문제 (최우선)
  2. 정답률이 낮은 문제
  3. 아직 안 나온 문제
  4. 나머지는 랜덤
  → 같은 시험 안에서 같은 문제가 처음부터 중복 출제되지 않게 한다 (오답 재출제는 예외).
- **답 입력**: 화면의 큰 숫자 키패드 (0~9, ⌫ 지우기, ✓ 확인). 키보드 입력도 지원.
- **정답 시**: 
  - 딩동 효과음 + 음성으로 영어 칭찬. 칭찬 문구는 최소 8종 랜덤: "Great job!", "Awesome!", "You got it!", "Brilliant!", "Super smart!", "Fantastic!", "You're on fire!", "Perfect!"
  - 화면에 ⭐ 획득 + 짧은 축하 애니메이션. 3연속 정답마다 콤보 이펙트(간단한 CSS 컨페티/이모지 팝).
- **오답 시**:
  - 부드러운 효과음 + 음성으로 격려: "Hmm, think again!", "Almost! Try once more!", "Give it another try!" 등 최소 5종 랜덤.
  - **같은 문제를 그 자리에서 1번 더 시도**하게 한다 (총 2회 기회).
  - 2회째도 틀리면 정답을 크게 보여주고 음성으로 읽어준다: "Seven times eight is fifty-six. Let's remember it!"
  - **틀린 문제는 이번 시험의 뒤쪽 큐에 다시 삽입**해서, 맞힐 때까지 시험이 끝나지 않는다. (재출제에서 맞히면 통과. 이것이 암기의 핵심 메커니즘.)
- **문제당 제한 시간 없음**, 단 답까지 걸린 시간을 기록한다 (3초 이내 정답 = "⚡ Fast!" 보너스 표시 — 유창성 동기부여용, 점수 불이익은 없음).

## 3. 결과 화면 (시험 종료 시)
- 점수: 첫 시도 기준 정답 수 / 20, 별점(⭐ 최대 3개: 20/20=3개+"PERFECT!", 18~19=2개, 15~17=1개).
- **틀렸던 문제 목록**을 카드로 크게 표시 (예: `7 × 8 = 56`) — 눈으로 한 번 더 복습.
- "Practice Missed Ones 🔁" 버튼: 틀린 문제만 다시 미니 퀴즈.
- "New Test 🚀" 버튼.
- 20/20 만점이면 화면 가득 컨페티 + 팡파레 효과음 + "PERFECT SCORE, SANGIL! 🏆" 음성.

## 4. 진행 상황 대시보드 (시작 화면에 포함)
- 큰 "START TEST 🚀" 버튼이 주인공.
- **오늘 완료한 시험 수**를 로켓/배지로 표시 (목표 5회: `🚀🚀🚀⚪⚪` 식). 날짜 바뀌면 리셋.
- **2~9단 마스터 현황**: 8×9 히트맵 그리드. 각 칸은 해당 문제의 숙련도 색 (회색=미출제, 빨강=자주 틀림, 노랑=보통, 초록=마스터: 최근 3회 연속 첫 시도 정답).
- 히트맵 아래 "Mastered: 54 / 72" 같은 진행률 바.

## 5. 디자인
- 9살 남자아이 취향: 밝고 경쾌한 우주/게임 테마 (로켓, 별, 그라데이션 배경). 촌스럽지 않게 — 둥근 카드, 부드러운 그림자, 큰 라운드 버튼.
- 문제는 화면 중앙에 아주 크게: `7 × 8 = ?` (숫자 최소 3rem 이상).
- 애니메이션은 CSS transition/keyframes로만. 정답/오답 피드백은 0.2초 내 즉각 반응.
- 이모지를 아이콘으로 적극 활용 (외부 이미지 금지이므로).
- 다크 배경 + 밝은 카드 조합 권장 (우주 테마와 어울림).

## 6. 완료 기준 (자체 검증 후 보고)
1. `index.html` 단일 파일로 완성, 더블클릭으로 열어서 전체 플로우 (시작 → 20문제 → 오답 재출제 → 결과 → 대시보드 갱신) 작동.
2. 오답 시나리오 검증: 일부러 틀렸을 때 재시도 → 정답 공개 → 뒤에서 재출제 → 맞히면 통과되는지.
3. localStorage 검증: 새로고침해도 오늘 시험 횟수·히트맵 유지, 날짜 변경 시 오늘 카운트만 리셋.
4. 음소거 토글 작동, 음소거 상태에서도 게임 진행에 문제 없는지.
5. 모바일 폭(375px)에서 키패드·문제·버튼이 잘리지 않는지.
