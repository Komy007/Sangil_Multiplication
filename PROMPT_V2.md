# 작업 지시서 V2: Sangil's Times Table Quest — 업그레이드

이 문서는 **이미 완성되어 작동 중인 `index.html`** (단일 파일 웹앱)을 업그레이드하는 지시서다.
V1 사양은 `PROMPT.md` 참조. **기존 기능을 깨뜨리지 말고** 아래 6가지를 추가/수정하라.

## 전제 (현재 구현 상태 — 반드시 코드를 읽고 파악할 것)
- 단일 `index.html`: 순수 HTML/CSS/Vanilla JS, 외부 리소스 금지 (변경 금지 원칙 유지)
- 화면: `screen-name`(이름 선택) → `screen-home`(대시보드) → `screen-quiz` → `screen-result` / `screen-admin`(학부모) / `screen-tetris`
- 프로필별 localStorage 네임스페이스: `nsKey(base, profile)` = `"sangil_" + base + "__" + encodeURIComponent(profile)`

- 효과음 Web Audio 합성, 음성 SpeechSynthesis(en-US), 🔊 음소거 토글
- 모든 UI 텍스트/음성은 영어 (사용자: 9세 영어 사용자)

## 1. 문제 형식 다양화 (지루함 방지 — 핵심)
현재는 `7 × 8 = ?` 한 가지 형식뿐. 시험 20문제 안에서 아래 형식이 **랜덤하게 섞여** 나오게 하라:

1. **기본형** `7 × 8 = ?` (전체의 약 50%)
2. **빈칸형(missing factor)** `7 × ? = 56` 또는 `? × 8 = 56` (약 25%)
   - 답은 빠진 인수. 곱셈↔나눗셈 감각을 동시에 키움.
3. **참/거짓형** `7 × 8 = 54  →  True? False?` (약 15%)
   - 큰 ✓/✗ 버튼 2개로 답함 (키패드 숨김). 오답 보기는 정답 ±(1~3) 또는 인접 구구단 값.
4. **역순형** `56 = ? × 8` (약 10%)

규칙:
- 어떤 형식이든 **내부 통계는 같은 fact(a×b) 기준**으로 기록한다 (기존 `factKey`/`recordFactResult` 재사용).
- 오답 재출제 시에는 **기본형으로** 다시 낸다 (암기 확정용).
- 형식별 안내는 불필요 — 화면만 보고 직관적으로 알 수 있게 크고 명확하게.
- 참/거짓형에서 "False"가 정답이면, 정답 공개 시 올바른 식을 보여준다: `7 × 8 = 56`.

## 2. 칭찬 문구 대폭 확충
- 정답 칭찬을 최소 **20종**으로 확대. 기존 8종 + 예: "Amazing, Sangil!", "You're a math star!", "Incredible!", "Boom! Nailed it!", "Genius mode ON!", "Rocket speed!", "That was lightning fast!", "High five! 🙌", "Unstoppable!", "Math wizard!", "Champion!", "Superb!" 등.
- 3연속/5연속/10연속 콤보 때는 **콤보 전용 문구**를 별도로 (예: "Three in a row!", "You're on a streak!", "TEN in a row?! LEGENDARY!").
- 같은 문구가 연속 2회 나오지 않게 (직전 문구 제외 후 랜덤).

## 3. 새 메뉴: 유럽식 세로셈 나눗셈 (European Long Division)
홈 화면에 **"Division Practice ➗"** 버튼을 추가하고 새 화면 `screen-division`을 만든다.

### 형식 (유럽/프랑스식 세로셈)
- 프랑스식 배치: 피제수(dividend)가 왼쪽, 제수(divisor)가 오른쪽 세로선 너머, 몫(quotient)은 제수 아래 가로선 밑에 쓴다:
```
  56 | 8
     |———
     | 7
```
- CSS로 이 레이아웃을 그린다 (세로선 + 가로선, monospace 큰 숫자).

### 문제 범위와 단계
- **Level 1 (기본)**: 구구단 역연산 — `56 ÷ 8` (2~9단 곱 결과 ÷ 인수, 나머지 0). 몫 한 자리.
- **Level 2 (도전)**: 두 자리 ÷ 한 자리, **나머지 있음** — 예: `47 ÷ 5 = 9 r 2`. 몫과 나머지(r) 두 칸 입력.
- 화면 상단에 Level 1 / Level 2 토글. 기본은 Level 1.

### 진행 방식
- 1세션 = 10문제. 기존 키패드 재사용 (Level 2는 몫 입력 → ✓ → 나머지 입력 → ✓ 순서, 현재 어느 칸을 입력 중인지 하이라이트).
- 정답/오답 처리, 칭찬/격려 음성, 2회 시도 후 정답 공개, 오답 뒤쪽 재출제 — **구구단 시험과 동일한 메커니즘** 재사용.
- Level 2 정답 공개 시 세로셈 풀이 과정을 단계별로 보여준다: `5 × 9 = 45`, `47 − 45 = 2` → `Quotient 9, Remainder 2`.
- 나눗셈 통계도 프로필별 localStorage에 저장 (`nsKey("divStats", profile)`), 관리자 대시보드에 나눗셈 세션 기록도 표시.
- 나눗셈 10/10 만점도 보너스 게임 자격을 준다 (아래 4·5항과 동일 규칙).

## 4. 테트리스 보드 크기 수정
- 현재 240×480 캔버스(10×20칸, 24px 블록)가 화면에서 짧아 블록이 너무 빨리 내려오는 느낌.
- **캔버스를 270×540 (블록 27px)으로 키우고**, 초기 낙하 간격을 500ms → **650ms**로 완화.
- 375px 폭에서 캔버스+조작버튼이 잘리지 않는지 확인 (캔버스 `max-width:100%` 유지, 필요시 세로 스크롤 허용).

## 5. 만점 보상 시간 누진제
- **오늘 하루 기준** 만점 횟수를 프로필별로 기록: `nsKey("perfectToday", profile)` = `{date, count}` (날짜 바뀌면 리셋).
- 보너스 게임 시간: 오늘 **1번째 만점 = 2분, 2번째 = 3분, 3번째 이후 = 5분**.
- 결과 화면 보너스 버튼에 시간을 표시: "🎮 Bonus Game (3 min)!"
- 게임 화면 타이머도 해당 시간으로 시작.

## 6. 벽돌깨기(Breakout) 추가 + 게임 선택
- 만점 시 보너스 버튼을 누르면 **게임 선택 화면**(또는 모달)이 뜬다: "🧱 Tetris" / "🎾 Brick Breaker" 큰 버튼 2개.
- Breakout 구현 (Canvas, 외부 리소스 금지):
  - 패들(하단), 공, 벽돌 6줄×8열 (줄마다 다른 색, 위 줄일수록 높은 점수).
  - 조작: 좌우 터치 버튼 + 캔버스 위 터치/드래그로 패들 이동 + 키보드 ←/→.
  - 공을 놓치면 목숨 1 감소 (총 3개, ❤️❤️❤️ 표시). 목숨 소진 시 벽돌/공 리셋하고 계속 (시간제라 게임오버 없음).
  - 벽돌 전부 깨면 새 판 + 보너스 점수. 벽돌 충돌음/패들 반사음은 Web Audio 합성.
  - 제한 시간(5항의 누진 시간) 종료 시 테트리스와 동일한 "Time's Up!" 오버레이 + 점수 + 홈 복귀.
- 두 게임 모두 시간제 공유, 음소거 준수.

5. 완료 기준 (자체 검증 후 보고):
   - 4가지 문제 형식이 실제로 섞여 출제되고, 형식별 정답/오답/재출제가 모두 작동
   - 나눗셈 Level 1/2 전체 플로우 + 세로셈 레이아웃이 유럽식으로 표시
   - 만점 1·2·3회째 보너스 시간이 2→3→5분으로 증가 (날짜 리셋 포함)
   - 테트리스/벽돌깨기 선택 화면, 두 게임 모두 시작·조작·시간종료·홈복귀 작동
   - 375px에서 모든 새 화면 잘림 없음
   - 기존 기능(프로필, 관리자 대시보드, 히트맵, 음소거) 회귀 없음
6. 검증은 브라우저 콘솔/스냅샷 기반으로 실제 플로우를 돌려서 확인하고, 결과를 사실대로 보고하라.
